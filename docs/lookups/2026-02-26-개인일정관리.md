# 개인 일정 등록이 귀찮아서 기억에 의존하는 문제

## 문제 정의
캘린더 앱에 일정을 등록하는 과정 자체가 귀찮아서 등록하지 않고 기억에 의존한다. 무의식에서 기억하려는 시도가 주의를 분산시킨다. 더 쉬운 채널로 일정을 등록하고, 적절한 때에 알림을 받고 싶다.

## 분석
- 사람 개입: 일정 등록은 사람 판단 필요, 알림은 자동
- 반복 주기: 비정기적
- AI 추론: 가벼운 수준 (자연어를 날짜/시간으로 파싱)
- 외부 서비스: 3개 (입력 채널, 일정 저장소, 알림 채널)

## 추천
**n8n 워크플로우**

입력 채널(메신저 등) → 일정 저장소(캘린더) → 알림 채널, 3개 서비스를 연동해야 하고, 자연어 입력을 날짜/시간으로 파싱하는 가벼운 AI 처리가 필요하다. n8n은 이런 멀티서비스 연동 + 가벼운 AI 조합에 가장 적합하다.

### 다른 방식이 안 되는 이유
- launchd + 셸 스크립트: 자연어 파싱과 3개 외부 서비스 연동이 필요한데 셸 스크립트로 다루기엔 복잡도가 과함. 이벤트 트리거 구현도 어려움
- Apple Shortcuts: 서비스 연동 수가 3개이고, 자연어 파싱 같은 AI 처리를 Shortcuts 내에서 자연스럽게 처리하기 어려움. 자동 알림 스케줄링도 제한적
- Raycast 확장: 로컬 키보드 호출 기반이라 다양한 채널에서의 입력을 받기 어려움. 서버 사이드 스케줄링(알림 시점 계산 후 자동 발송)을 담당할 수 없음
- Claude Code 스킬: 깊은 AI 추론이 아닌 가벼운 파싱 수준이므로 과함. 상시 대기하며 알림을 보내는 자동화에 적합하지 않음
