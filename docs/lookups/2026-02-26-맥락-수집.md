# AI 작업을 위한 개인 맥락 자동 수집

## 문제 정의
AI에게 작업을 맡길 때 나의 전체 맥락(직무, 진행 중인 프로젝트, 처리 중인 이슈, 해왔던 일, 해야 할 일)이 주입되지 않아 원하는 결과를 얻지 못한다. 이번에는 "맥락 주입"은 제외하고, "맥락 수집"에만 초점을 맞춘다.

## 분석
- 사람 개입: 필요 없음 — 자동으로 수집되면 됨
- 반복 주기: 특정 이벤트 발생 시 — GitHub 커밋/PR/이슈, Linear 이슈 변경 등
- AI 추론: 불확실하나 가벼운 수준 — 수집 자체는 단순 파이프라인, AI 활용 여지는 있음
- 외부 서비스: Linear, GitHub (1-2개)

## 추천
**n8n 워크플로우**

맥락 수집은 사람 개입 없이 이벤트 기반으로 외부 서비스(Linear, GitHub)에서 데이터를 자동 수집하는 문제다. n8n은 웹훅과 API 연동으로 Linear 이슈 변경, GitHub 커밋/PR/이슈 이벤트를 감지하여 맥락을 자동으로 수집·저장하는 데 최적화되어 있다.

### 다른 방식이 안 되는 이유
- launchd + 셸 스크립트: 정기 스케줄에는 적합하나, 이벤트 기반 트리거(웹훅 수신)를 처리하기 어려움. 두 서비스의 API 연동을 셸에서 관리하는 것은 비효율적.
- Apple Shortcuts: 디바이스/위치 트리거 기반. Linear나 GitHub 이벤트를 트리거로 사용 불가.
- Raycast 확장: 사용자가 매번 수동 호출 필요. "자동 수집" 요구사항에 부적합.
- Claude Code 스킬: 대화형 AI 추론에 적합. 맥락 수집은 이벤트 감지→데이터 수집→저장의 단순 파이프라인이므로, 세션 기반인 Claude Code보다 상시 가동되는 n8n이 적합.
