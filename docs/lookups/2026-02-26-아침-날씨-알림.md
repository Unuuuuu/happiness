# 매일 아침 날씨 정보(비 여부, 최저/최고 온도)를 핸드폰으로 받기

## 문제 정의
아침에 오늘 비가 올지, 최저 온도와 최고 온도가 어떻게 되는지 핸드폰으로 알림을 받고 싶다. 어떤 채널(Slack, Telegram, 카카오톡 등)로 접근할지도 결정이 필요하다.

## 분석
- 사람 개입: A) 필요 없음 — 자동으로 돌아가면 됨
- 반복 주기: B) 매일 정기적 스케줄
- AI 추론: A) 불필요 — 단순 반복/자동화
- 외부 서비스: B) 1-2개 서비스 (날씨 API + 알림/메시징 서비스)

## 추천
**n8n 워크플로우**

사람 개입 없이 매일 정해진 시간에 외부 서비스(날씨 API + 알림 서비스)를 연동해 자동으로 메시지를 보내는 구조이므로, n8n 워크플로우가 가장 적합하다.

### 다른 방식이 안 되는 이유
- **launchd + 셸 스크립트**: 정기 스케줄에는 맞지만, 외부 API 호출과 메시징 서비스 연동을 셸 스크립트로 관리하면 인증 처리, 에러 핸들링이 번거로움
- **Apple Shortcuts**: 자동화 가능하나, 백그라운드 스케줄 실행의 신뢰성이 낮고 서버 없이 iPhone이 켜져 있어야 함
- **Raycast 확장**: 사람이 직접 호출하는 구조라 "매일 아침 자동 전송"에 맞지 않음
- **Claude Code 스킬**: AI 추론이 불필요한 단순 데이터 조회+전달이므로 과잉

## 다음 단계
n8n에서 워크플로우를 구성한다:
1. **트리거**: Schedule 노드로 매일 아침 원하는 시간에 실행
2. **날씨 조회**: HTTP Request 노드로 날씨 API(기상청 API, OpenWeatherMap 등) 호출
3. **메시지 포맷팅**: 비 여부, 최저/최고 온도를 보기 좋게 가공
4. **알림 전송**: 선호하는 채널(Slack, Telegram, 카카오톡 등)로 메시지 전송

`n8n-workflow-patterns`, `n8n-node-configuration` 스킬을 활용하여 워크플로우를 설계할 수 있다.
