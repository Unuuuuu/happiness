# 맥락 수집 시스템에 과거 데이터가 없어 /work-scan이 유의미한 분석을 할 수 없다

## 문제 정의

맥락 수집 시스템(Linear/GitHub 웹훅 → PostgreSQL)이 오늘 가동되어, 웹훅은 앞으로의 이벤트만 수신한다. 시스템 가동 이전의 Linear 이슈 상태와 GitHub 활동이 수집되지 않아 /work-scan이 의미 있는 패턴 분석을 할 수 없다.

## 분석

- 사람 개입: 불필요 — 자동으로 돌아가면 됨
- 반복 주기: 일회성 — 한 번만 채우면 됨
- AI 추론: 불필요 — API에서 데이터를 가져와 DB에 넣는 단순 작업
- 외부 서비스: 3개 이상 — Linear API, GitHub API, PostgreSQL

## 추천

**n8n 워크플로우**

일회성이지만 Linear API, GitHub API, PostgreSQL 3개 서비스 연동이 필요하고, 사람 개입이나 AI 추론 없이 자동으로 데이터를 가져와 저장하는 작업이다. n8n은 이미 맥락 수집에 사용 중이며, HTTP Request + PostgreSQL 노드를 조합하여 API 페이지네이션과 UPSERT를 처리하기에 적합하다.

### 다른 방식이 안 되는 이유

- launchd + 셸 스크립트: 3개 외부 서비스 API 연동(인증, 페이지네이션, 에러 처리)을 셸 스크립트로 처리하면 복잡도가 높아짐
- Apple Shortcuts: 서버사이드 API 호출 + DB 쓰기 작업에 적합하지 않음
- Raycast 확장: 키보드 기반 즉석 호출용이며, 백그라운드 데이터 마이그레이션에 맞지 않음
- Claude Code 스킬: AI 추론이 불필요한 단순 데이터 수집/저장 작업으로 과잉 설계
