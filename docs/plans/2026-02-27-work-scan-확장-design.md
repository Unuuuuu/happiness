# /work-scan 기간 확장 설계

## 목표

`/work-scan`에 기간 인자를 지원하고, 장기 데이터에 맞는 패턴 분석을 추가한다.

## 현재 동작

- 기본 14일, 데이터 없으면 30일로 확장
- 패턴: 장기 체류, 우선순위 불일치, 리뷰 대기, 활동 공백, 반복 키워드

## 변경 사항

### 1. 기간 인자 지원

- `/work-scan` → 기본 14일 (기존과 동일)
- `/work-scan 90` → 최근 90일
- `/work-scan 180` → 최근 180일
- ARGUMENTS에서 숫자를 파싱하여 `days` 파라미터로 사용
- 데이터 없을 때 자동 확장 로직은 유지 (14→30, 사용자 지정 시 해당 값 그대로 사용)

### 2. 장기 패턴 분석 (30일+ 데이터일 때 추가)

기존 5가지 패턴은 유지하고, 데이터 기간이 30일 이상일 때 추가 패턴을 탐지:

- **반복 작업**: 비슷한 유형의 이슈가 반복 등장 (예: 같은 종류의 고객 문의가 여러 건)
- **완료 후 재등장**: Done 처리된 주제가 새 이슈로 다시 나타남
- **프로젝트 집중도 변화**: 특정 프로젝트에 활동이 몰렸다 사라지는 패턴
- **상태 순환**: 이슈가 상태를 왔다갔다 하는 패턴 (In Progress ↔ Todo 등)

### 3. 현황 요약 조정

기간이 길 때는 요약에 시간 분포 정보를 추가:
```
### 현황 요약 (90일)
- Linear: N건 (In Progress M건, Todo K건, Done J건)
- GitHub: PR N건, Push M건
- 활동 프로젝트: project1, project2
- 활동 집중 시기: 최근 2주에 N건, 그 전 76일에 M건
```

## 변경하지 않는 것

- HARD-GATE 규칙 전부 유지
- 실행 흐름 (데이터 수집 → 현황 요약 → 패턴 분석 → 제안) 유지
- surface 연결 방식 유지
- 최대 3개 제안 제한 유지
