# 오디오 학습 파이프라인 — 큰 그림

## 목적

영어 콘텐츠를 **품질 좋은 한국어 음성**으로 변환하여, 이동 중이나 휴식 중에 귀로 들으며 지식 수준을 높인다.

## 핵심 흐름

```
소스 입력 → 콘텐츠 추출 → 핵심 정리 → 음성 합성 → 소비
```

## MECE 분해

### 1. 소스 입력 (Source Ingestion)

외부 콘텐츠를 시스템에 넣는 단계.

| 소스 유형 | 예시 |
|---|---|
| 웹 문서 | 블로그, 아티클, 기술 문서 |
| 영상 | YouTube 등 자막/음성이 있는 영상 |

### 2. 콘텐츠 추출 (Content Extraction)

소스에서 학습 가능한 텍스트를 뽑아내는 단계.

- 웹 문서 → 본문 텍스트 추출 (광고, 네비게이션 등 노이즈 제거)
- 영상 → 자막/트랜스크립트 추출

### 3. 핵심 정리 (Knowledge Distillation)

추출된 원문을 **학습에 최적화된 한국어 콘텐츠**로 변환하는 단계. 이 파이프라인의 핵심 가치.

- 구체적이되 핵심만 담은 내용 구성
- 자연스러운 한국어 의역 (귀로 들었을 때 편한 문체)
- 원문의 지식 밀도를 유지하면서 불필요한 서술 제거

### 4. 음성 합성 (Text-to-Speech)

정리된 한국어 텍스트를 음성 파일로 변환하는 단계.

- 한국어 TTS 엔진 선택 및 음성 생성
- MP3 등 범용 포맷으로 출력

### 5. 소비 (Delivery & Consumption)

생성된 음성을 실제로 듣는 단계.

- MP3 파일 저장 (오프라인 재생, 팟캐스트 앱 등)
- Telegram 음성 메시지 전송 (즉시 듣기)

## 설계 원칙

- **핵심 정리(3단계)가 품질을 결정한다** — TTS 품질보다 콘텐츠 정리 품질이 학습 효과에 더 큰 영향
- **각 단계는 독립적으로 교체 가능해야 한다** — 소스 유형 추가, TTS 엔진 교체 등이 다른 단계에 영향을 주지 않아야 함
- **점진적으로 구축한다** — 모든 소스/모든 기능을 한번에 만들지 않고, 하나의 소스(웹 문서)부터 파이프라인을 관통시킨 뒤 확장
