# 일을 시작하기 전에 정의·등록하는 단계가 마찰 때문에 건너뛰어진다

## 발견 과정
일/업무 영역에서 반복되는 불편을 탐색하던 중, 사용자가 일의 워크플로우가 일관되지 않다는 문제를 직접 제기함. `/problem-definition` 스킬로 문제를 깊이 파고들어 근본 원인을 구체화함.

## 문제 정의
- 영역: 일/업무
- 빈도: 매일
- 고통 정도: 짜증 + 시간 낭비 + 실질적 손해 (모든 수준 해당)
- 현재 대처: 간단한 투두리스트를 시도하지만 관리가 잘 되지 않음

### Struggling Moment
일이 생겼을 때 (아이디어, 요청, 발견 등), 바로 손이 나가서 시작하게 됨. "이 일이 뭔지, 왜 하는지" 정의하고 추적 시스템에 등록하는 단계가 건너뛰어진다.

### 마찰의 원인 (2가지)
1. **도구 마찰** — 도구를 열고, 프로젝트 찾고, 이슈 만드는 과정의 체감 비용
2. **인지 마찰** — 제목과 설명을 뭐라고 써야 할지 생각하는 체감 비용

### 핵심 통찰
실제 등록 비용은 크지 않다. 체감 비용이 문제다. **시스템을 만드는 게 아니라 마찰을 없애는 게 핵심이다.**

### 이로 인한 고통 (5가지)
1. **기록 부재** — 뭘 했는지 남지 않음
2. **추적 불가** — 해야 할 일들이 시스템이 아닌 머리에만 있음
3. **기억 부담** — 계속 머리로 들고 있어야 함
4. **구조 부재** — 일이 구조화되지 않아 정신적 산만함 발생
5. **우선순위 부재** — 관리가 안 되니 뭘 먼저 할지 판단도 안 됨
