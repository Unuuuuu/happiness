# 모든 일의 수명주기가 하나의 시스템에서 일관되게 추적되지 않는다

## 발견 과정
일/업무 영역에서 반복되는 불편을 탐색하던 중, 사용자가 일의 워크플로우가 일관되지 않다는 문제를 직접 제기함. 일을 시작하기 전에 정의와 우선순위 판단이 필요하고, 하나의 진실된 관리 저장소에서 상태가 추적되어야 하는데 그렇지 못하다는 점을 구체적으로 설명함.

## 문제 정의
- 영역: 일/업무
- 빈도: 매일
- 고통 정도: 짜증 + 시간 낭비 + 실질적 손해 (모든 수준 해당)
- 현재 대처: 간단한 투두리스트를 시도하지만 관리가 잘 되지 않음
- 배경: 두 가지 핵심 고통점이 동시에 존재한다. (1) 일을 정의/추적하기 전에 바로 작업에 뛰어드는 습관 — 정리보다 실행이 먼저 나감. (2) 중간에 우선순위가 바뀌거나 일을 중단/취소할 때 상태가 적절히 반영되지 않음. 하나의 진실된 관리 저장소(single source of truth)가 없어서 일의 상태(시작, 진행, 중단, 취소, 완료)가 흩어지고, 정신적 산만함까지 이어지는 상태.
